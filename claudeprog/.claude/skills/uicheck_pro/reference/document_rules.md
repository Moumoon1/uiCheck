# Document Rules

## 文档目标
文档要：
- 让设计师快速复核
- 让前端快速定位
- 让产品快速看到问题状态
- 方便跟进和验收

## 默认结构
1. 标题区
2. 走查摘要
3. 页面总览对照
4. 问题表格

## 问题表格表头
固定为：
- 序号
- 问题描述
- 状态
- 问题截图
- 日期
- 优先级
- 走查人
- 跟进人

## 列内容说明

### 问题描述
只写：
- 问题
- 建议

### 状态
只允许：
- 待修改
- 待验收
- 已验收

### 问题截图
- 左：开发页局部图
- 右：设计稿局部图
- 两边都框同一个问题对象

### 日期
默认写文档生成日期。

### 优先级
只写：
- P0
- P1
- P2

### 走查人
用户没提供时默认：
- AI 走查助手

### 跟进人
用户没提供时默认：
- 待指定

## 版式建议
- 问题描述列和问题截图列更宽
- 状态、日期、优先级列更窄
- 跨页时重复表头
- 单元格允许换行
