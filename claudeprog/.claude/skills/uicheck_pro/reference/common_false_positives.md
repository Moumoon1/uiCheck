# Common False Positives

以下内容默认不作为正式问题：

## 1. 动态数据差异
- 昵称不同
- 金额不同
- 时间不同
- 列表条数不同
- 运营配置文案不同

前提：结构仍然成立。

## 2. 截图条件差异
- 滚动位置不同
- 截图长度不同
- 缩放比例不同
- 图片压缩不同

## 3. 系统层差异
- 状态栏时间不同
- Wi-Fi / 电量图标不同
- 系统默认字体渲染轻微不同

## 4. 低价值细节
- 1~2px 级别的上下偏移
- 个别小字号差异
- 个别小间距差异
- 极弱的视觉感觉型判断

## 5. 模板污染信号
- 问题描述复用了 reference 中现成的业务名或模块名
- 结论里出现截图中看不到的对象名称
- 同一批结果多条问题使用高度相似的模板句

## 默认处理建议
- 证据稳定：进正式问题
- 有迹象但不稳定：进疑似问题
- 价值低且证据弱：不纳入问题
- 若怀疑模板污染：回查读图结果、运行时 prompt 和最终输出的一致性
