# Examples

这些示例只用于说明写法结构，不代表当前任务的真实结论。

## 正式问题写法模板
问题：开发页【某模块 / 某元素】与设计稿在【结构 / 层级 / 对齐 / 样式】上不一致。  
建议：按设计稿恢复【对应模块 / 元素】的【结构 / 样式 / 位置关系】。

## 疑似问题写法模板
问题：开发页【某局部细节】可能与设计稿存在差异。  
建议：建议对【对应区域】做局部放大、同尺寸比对或叠图确认。

## 错误写法示例
标题：这里有点问题  
位置：大概在右上区域  
问题：我觉得整体看上去不太对，可能有一些地方没有完全还原。  
影响：会影响整体感受。  
建议：建议优化。

问题：
- 太空
- 太泛
- 不可执行
- 把位置和感受混进问题描述

## 使用限制
- 不要把本文件中的占位词替换成截图里不存在的业务对象
- 不要把本文件中的模板句直接当作最终结论
- 最终输出必须改写为基于当次截图证据的表述
