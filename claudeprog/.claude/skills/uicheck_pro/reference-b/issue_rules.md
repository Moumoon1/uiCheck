# Issue Rules（B端页面）

## 正式问题
正式问题要满足：
- 差异明确稳定
- 不太可能由截图条件造成
- 可以直接提给开发修改

常见正式问题类型（B端）：
- 表格列缺失或列顺序错误
- 表头标签名称不一致
- 筛选栏 / 搜索框缺失或结构不对
- 分页器样式、位置或格式不一致
- 操作按钮组缺失、顺序错误或样式不对
- Tab / 标签页切换项不一致
- 表单字段缺失或校验提示缺失
- 弹窗 / 下拉内容结构不一致
- 模块层级 / 顺序明显错误
- 关键状态标签（如状态 badge、标签颜色）不一致
- 表格行高 / 列宽明显偏差
- 切图/裁剪错误（卡片或图片边缘被截断、元素溢出被裁掉）
- 按钮 / 图标 / 输入框 / 标签 / 下拉框 / 单选 / 多选 组件样式明显不对
- 面包屑层级缺失或顺序错误
- 图表内部结构差异（折线样式/数据点/网格线/坐标轴/图例/填充等）

## 疑似问题
疑似问题用于记录：
- 看起来有偏差
- 对 B端信息密度和操作效率重要
- 但暂时不适合直接作为"必须修改"的正式问题

常见疑似问题类型（B端）：
- 表格行高疑似偏大或偏小
- 筛选区高度或间距疑似不一致
- 按钮间距疑似偏窄或偏宽
- 表头背景色或边框线疑似有偏差
- 字段标签对齐方式疑似不同
- 分割线 / 分组线疑似缺失或多余
- 空态 / 加载态展示疑似缺失
- 图表内部细节（网格/数据点/填充）疑似有偏差但不确定

## 问题描述怎么写

### 通用精准描述原则（所有问题类型必须遵守）

**problem 字段必须包含两侧可见的具体证据，不允许只描述差异类型：**
- 不合格：`Tab 项不一致` / `按钮样式不对` / `间距偏小` / `图表不一致`
- 合格：`开发稿 [具体元素] 为 [实际值]，设计稿为 [期望值]`

---

### 各类问题精准描述格式

#### 枚举 / 列表类（只需指出差异项，不必全列两侧所有项）

**核心原则**：只描述缺少的、多余的、或错误的具体项名，不需要把两侧所有项都列出来。

**Tab / 标签页**
- 格式：`开发稿 Tab 缺少"[具体项名]"` 或 `开发稿 Tab 多出"[具体项名]"` 或 `开发稿 Tab "[原名]" 应为 "[设计稿名]"`
- 不合格：`Tab 项不一致`
- 不合格（过度）：`设计稿 Tab 含"全部、已发布、审核中"三项，开发稿含"全部、已发布"两项，缺少"审核中"`
- 合格：`开发稿 Tab 缺少"审核中"`

**表格列**
- 格式：`开发稿缺少"[列名]"列` 或 `开发稿多出"[列名]"列` 或 `开发稿"[列名]"列顺序有误，应在"[相邻列名]"[前/后]`
- 不合格：`列缺失`
- 合格：`开发稿缺少"金额"列`

**操作按钮组**
- 格式：`开发稿操作列缺少"[按钮名]"` 或 `开发稿操作列多出"[按钮名]"` 或 `开发稿"[按钮名]"顺序有误`
- 不合格：`操作按钮缺失`
- 合格：`开发稿操作列缺少"详情"按钮`

**筛选区**
- 格式：`开发稿筛选区缺少"[控件名]"` 或 `开发稿筛选区多出"[控件名]"`
- 合格：`开发稿筛选区缺少"时间范围选择器"`

**面包屑**
- 格式：`开发稿面包屑缺少"[层级名]"层级` 或 `开发稿面包屑顺序有误`
- 合格：`开发稿面包屑缺少"活动管理"层级`

**单选 / 多选选项**
- 格式：`开发稿 [单选/多选] 缺少"[项名]"` 或 `开发稿多出"[项名]"`
- 合格：`开发稿单选组缺少"月"`

**下拉框选项**
- 格式：`开发稿 [下拉名称] 缺少"[选项名]"` 或 `多出"[选项名]"`
- 合格：`开发稿状态下拉缺少"草稿"选项`

---

#### 组件样式类（必须描述两侧各自的具体样式，不允许只说"不对"）

**按钮样式**
- 格式：`设计稿 [主/次/危险] 按钮为 [填充/边框/文字] + [颜色描述]；开发稿为 [描述]`
- 不合格：`按钮样式不对`
- 合格：`设计稿主按钮为实色蓝色填充+白字，开发稿为透明背景蓝色边框+蓝字（幽灵按钮样式）`

**按钮尺寸**
- 格式：`设计稿按钮高度约 [X]px，开发稿约 [Y]px，偏 [大/小] 约 [比例]%`
- 合格：`设计稿主按钮高度约 36px，开发稿约 28px，偏小约 22%`

**图标**
- 格式：`设计稿 [操作名] 操作 [有/无] 图标 + [图标描述]；开发稿 [有/无/不同]`
- 不合格：`图标缺失`
- 合格：`设计稿"编辑"操作有铅笔图标+文字，开发稿只有文字无图标`

**输入框**
- 格式：`设计稿输入框为 [圆角/直角] + [有/无填充背景] + [边框描述]；开发稿为 [描述]`
- 合格：`设计稿输入框为圆角 8px 带灰色填充背景，开发稿为直角无填充透明背景边框样式`

**标签 / Badge**
- 格式：`设计稿 "[状态文字]" 为 [填充/边框] + [颜色] 样式；开发稿为 [描述]`
- 不合格：`状态标签颜色不对`
- 合格：`设计稿"上架"状态为绿色填充标签，开发稿为无填充绿色边框样式`

**下拉框外观**
- 格式：`设计稿下拉框 [有/无] 箭头图标 + [圆角/直角] + [边框/背景描述]；开发稿为 [描述]`
- 合格：`设计稿下拉框右侧有向下箭头图标，开发稿缺少箭头图标`

**单选控件**
- 格式：`设计稿单选按钮为 [圆形/方形] + [选中样式描述]；开发稿为 [描述]`
- 合格：`设计稿单选按钮选中状态为实心圆+品牌色，开发稿为方框勾选样式（应为圆形）`

**多选控件**
- 格式：`设计稿多选框为 [方形/圆角方形] + [选中样式描述]；开发稿为 [描述]`
- 合格：`设计稿多选框选中为蓝色填充+白色勾，开发稿选中为蓝色边框+无填充`

---

#### 量化类（必须给出估值或相对比较，禁止只说"偏大/偏小"）

**间距 / 内边距**
- 格式：`设计稿 [区域] 间距约 [X]px，开发稿约 [Y]px，偏 [大/小] 约 [比例]%`
- 不合格：`间距偏小`
- 合格：`设计稿表格行间距约 16px，开发稿约 8px，偏小约 50%`

**行高**
- 格式：`设计稿表格行高约 [X]px，开发稿约 [Y]px，偏 [大/小] 约 [比例]%`
- 合格：`设计稿表格行高约 48px，开发稿约 36px，偏小约 25%`

**字号**
- 格式：`设计稿 [元素] 字号约 [X]px，开发稿约 [Y]px，差约 [Z]px`
- 不合格：`字号不同`
- 合格：`设计稿表头字号约 14px，开发稿约 12px，差约 2px`

**字重**
- 格式：`设计稿 [元素] 为 [加粗/常规]，开发稿为 [加粗/常规]`
- 合格：`设计稿页面标题为加粗，开发稿为常规字重`

**信息层级**
- 格式：`设计稿 [元素] 约 [字号]px+[字重]（[级别]级视觉权重），开发稿降为约 [字号]px+[字重]，层级被压缩`
- 合格：`设计稿页面标题约 18px 加粗（视觉一级），开发稿降为约 14px 常规（与正文同级），层级被压缩`

**对齐**
- 格式：`设计稿 [元素] [左/右/居中] 对齐，开发稿为 [对齐方式]`
- 合格：`设计稿操作按钮组右对齐，开发稿居中对齐`

---

#### 图表内部结构类（禁止只看图表外形，必须检查内部细节）

> **重要**：图表外形和类型与设计稿一致，不代表内部结构一致。必须逐项检查以下内容：

**折线 / 曲线样式**
- 格式：`设计稿折线为 [实线/虚线/点线] + 粗细约 [X]px；开发稿为 [描述]`
- 合格：`设计稿折线为虚线，开发稿为实线`

**数据点标记**
- 格式：`设计稿折线节点为 [实心圆/空心圆/菱形/无]；开发稿为 [描述]`
- 合格：`设计稿折线节点为实心圆，开发稿无数据点标记`

**网格线**
- 格式：`设计稿有 [横向/纵向] [实线/虚线] 网格线；开发稿 [有/无/样式不同]`
- 合格：`设计稿有横向虚线网格，开发稿无网格线`

**坐标轴标签 / 单位**
- 格式：`设计稿 [X/Y] 轴有 [单位/标注]；开发稿 [缺少/不同]`
- 合格：`设计稿 Y 轴有单位标注"万元"，开发稿无单位`

**图例样式**
- 格式：`设计稿图例为 [圆点/方块/线段] + 文字；开发稿为 [描述]`
- 合格：`设计稿图例为圆点+文字，开发稿为方块+文字`

**填充区域**
- 格式：`设计稿折线下方有 [渐变/纯色] 填充；开发稿 [有/无/不同]`
- 合格：`设计稿折线下方有蓝色渐变填充，开发稿无填充`

**柱体细节**
- 格式：`设计稿柱体为 [圆角/直角] + [颜色/渐变描述]；开发稿为 [描述]`
- 合格：`设计稿柱体顶部为圆角，开发稿为直角`

**参考线 / 目标线**
- 格式：`设计稿有 [颜色/样式] 参考线；开发稿缺少`
- 合格：`设计稿有红色横向虚线目标参考线，开发稿缺少`

**数据标注**
- 格式：`设计稿 [折线/柱顶] 有 [数值/百分比] 标注；开发稿 [有/无]`
- 合格：`设计稿柱顶有数值标注，开发稿无`

---

## 不要这样写
- 写成长段小作文
- 在问题描述里重复写位置
- 在问题描述里重复写优先级和状态
- 在问题描述里塞很多背景解释
- 直接复制示例句子作为本次结论
- 只描述差异类型而不说具体是什么（"不一致"、"缺失"、"不对"）
- 对图表只检查外形和数量，不检查内部细节
