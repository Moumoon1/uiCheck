# Common False Positives（B端页面）

以下内容默认不作为正式问题：

## 1. 动态数据差异
以下属于接口返回的动态数据，**禁止报为任何问题（confirmed 或 suspected）**：
- 表格中的具体数值（金额、数量、百分比、转化率）
- 日期 / 时间字段的具体内容
- 用户名、头像、ID
- 排名、序号
- 图表数据值（柱高、折线点位、饼图比例）

前提：结构和标签仍然成立。**如果是表头标签名称不同、列缺失、筛选选项文案不同——这些是设计稿固定定义的，不属于动态数据，应正常上报。**

## 2. 截图条件差异
- 滚动位置不同
- 截图长度不同
- 缩放比例不同
- 图片压缩不同

## 3. 系统层差异
- 状态栏时间 / Wi-Fi / 电量图标不同
- 系统默认字体渲染轻微不同

## 4. 低价值细节
- 1~2px 级别的上下偏移（**注意：如果是卡片/表格边缘被明显裁切，不属于此类，应作为问题上报**）
- 个别小字号差异（<2px 且视觉上不明显）
- 个别小间距差异

注意：B端表格和列表的行高、列宽、间距差异如果影响信息密度或操作效率，即使幅度不大也应纳入疑似问题。

## 5. 模板污染信号
- 问题描述复用了 reference 中现成的业务名或字段名
- 结论里出现截图中看不到的对象名称
- 同一批结果多条问题使用高度相似的模板句

## 6. 文案相同性核查（防幻觉）

**任何 problem 含"文案不同"、"标签名不同"、"文字不一致"、"文字有差异"等表述，必须执行以下流程：**

1. 在截图中逐字读取两侧实际可见文字
2. 在 problem 字段中使用格式：`开发稿显示"[实际文字]"，设计稿显示"[实际文字]"`
3. 若两侧文字完全相同 → **必须删除该条问题，不得输出**
4. 若因字体渲染 / 图片压缩无法确认是否相同 → 进 suspected，不进 confirmed
5. 禁止基于"感觉字体不同"、"看起来不一样"、"字形疑似有差异"输出文案差异结论

**压缩失真判断标准**（当文字边缘模糊时）：
- 判断主干笔画形态是否不同（而非边缘锯齿）
- 判断字形整体结构是否不同
- 若仅边缘模糊但主体可辨认为相同字 → 不得报为文案差异

## 7. 视觉量化幻觉防御（间距 / 字号 / 行高）

**间距 / 字号 / 行高问题进 confirmed，必须满足以下至少一条：**
- 提供视觉估值（如"约是相邻元素高度的 1.5 倍 vs 设计稿的 2 倍"）
- 有明显可辨的视觉差异（不是 1~3px 的细微差）
- 两侧比例关系明显不同（如一侧紧凑，另一侧宽松）

**若仅凭"感觉偏小/偏大"无法量化 → 进 suspected，不进 confirmed**

禁止行为：
- 不得仅凭模糊感知就在 confirmed 里报间距/字号问题
- 不得把因宽度缩放导致的整体比例变化误报为"间距偏小"

## 8. 图表"外形一致"不等于"内部结构一致"（防漏报）

**禁止因"图表外形和类型一致"就跳过内部检查。**

即使图表种类、数量、位置与设计稿一致，仍必须逐项检查：
- 折线 / 曲线：实线 vs 虚线 vs 点线，粗细是否一致
- 数据点标记：有无实心圆 / 空心圆 / 无标记
- 填充区域：折线下方有无渐变或纯色填充
- 网格线：有无横向 / 纵向网格，实线 vs 虚线
- 坐标轴标签：X / Y 轴标注是否完整，单位是否存在
- 图例：样式（圆点 / 方块 / 线段）+ 文字是否完整
- 参考线 / 目标线：有无特殊标注线
- 柱体细节：圆角 vs 直角，渐变 vs 纯色
- 数据标注：折线节点 / 柱顶是否有数值标注

**只有上述全部核查后，才可标记为"图表结构一致"。若存在任何可见差异，必须上报。**

---

## 默认处理建议
- 证据稳定：进正式问题
- 有迹象但不稳定：进疑似问题
- 价值低且证据弱：不纳入问题
- 若怀疑模板污染：回查读图结果、运行时 prompt 和最终输出的一致性
