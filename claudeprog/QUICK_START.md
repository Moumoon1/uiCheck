# 快速开始指南

## 🚀 3 步开始使用

### 第 1 步：启动服务

**使用 Claude Code / Codex**（推荐）：
```
启动设计师平台
```

**或手动启动**：
```bash
cd designer-platform
npm start
```

### 第 2 步：上传截图

访问 http://localhost:3000，然后：

1. 选择页面类型（C端/B端）
2. 上传开发稿截图（命名为 `dev_xxx.png`）
3. 上传设计稿截图（命名为 `design_xxx.png`）
4. 选择视觉模型（推荐 GPT 5.4）
5. 点击"开始走查"

### 第 3 步：查看结果

等待分析完成，查看：
- **确认问题**：必须修复的问题
- **疑似问题**：需人工确认的问题
- **问题截图**：带标注的对比图

## 🎯 核心概念

### 页面类型

- **C端页面**：移动端活动页、功能页（默认）
- **B端页面**：管理后台、数据看板

### 视觉模型

| 模型 | 推荐场景 | 注意事项 |
|------|----------|----------|
| GPT 5.4 | 所有场景（推荐） | 平衡性能和额度 |
| Kimi K2.5 | 中文页面 | 速度快，稳定 |
| Claude Sonnet 4.6 | 复杂细节对比 | 有额度限制 |
| Gemini 3.1 Pro | 多语言页面 | 有额度限制 |

### 问题优先级

- **P0**：严重问题，必须立即修复
- **P1**：重要问题，建议修复
- **P2**：次要问题，可选修复

## 💡 使用技巧

1. **命名规范**：使用清晰的文件名（如 `dev_首页.png`）
2. **背景信息**：说明走查重点，帮助 AI 理解
3. **模型选择**：首次使用推荐 GPT 5.4
4. **结果使用**：优先处理 P0 和 P1 问题
